How FairScale Agent Trust Works

FairScale's Agent Trust Directory scores every AI agent on Solana by combining on-chain behavioral analysis with protocol verification and social reputation. The result is a single trust score — the Agent FairScore — that protocols can embed in their transaction flows to decide whether to interact with an agent.

What is the Agent FairScore?

A composite score from 0-100 blending FairScale's neural network wallet analysis (trained on 13M+ transactions, rescored every 24 hours) with our six-pillar trust interpretation layer. The formula:

Score = (FairScale Base × 20%) + (Trust Composite × 80%) + Multi-Registry Bonus + Attestation Boost + Description Bonus − Red Flag Penalties

The trust composite is weighted heavily because verification and behavioral analysis are the strongest trust indicators — a raw wallet score alone isn't enough to assess an AI agent.

The Six Trust Pillars

Verification (35-40%) — the dominant signal. Protocol registrations on SAID, ERC-8004, and SATI prove commitment. A single registry scores 40/100. Dual-registry scores 65/100. Triple: 75/100. ClawKey human verification adds +20. SAID reputation tier adds up to +12.

Reliability (22-25%) — behavioral consistency: conviction ratio, transaction tempo, burst patterns, capital flow direction. Agents that dump tokens or drain SOL get penalized.

Social & Reputation (0-12%) — FairScale social_score, SAID reputation score, SAID attestations/feedback, and linked social handles. If no social data is available, this pillar's weight redistributes to Verification and Reliability automatically.

Track Record (13-16%) — wallet age, active days, activity consistency, median hold duration, and transaction cadence. FairScale Pro provides 1-year behavioral windows.

Economic Stake (8-9%) — stablecoins, native SOL, liquid staking tokens, and major token positions.

Ecosystem (10%) — cross-protocol diversity. An agent active across 8 protocols is more trustworthy than one locked to a single contract.

Multi-Registry Bonus

Agents verified on multiple protocols receive a direct score bonus outside the pillar system: +10 for dual registry (e.g. SAID + ERC-8004), +15 for triple. This is the single strongest trust signal — an agent that independently registered on multiple protocols demonstrates real commitment.

Attestation Network

SAID reputation feedback and SATI/SAS on-chain attestations are weighted by the attester's own trustworthiness (quadratic weighting). An endorsement from a 90-score agent is worth 9× more than one from a 30-score agent. High-quality attestation networks boost scores by up to +15 points.

Description Quality

Agents with detailed, specific descriptions about their capabilities earn a bonus (up to +5 points). The system analyzes for professional keywords, specific protocol mentions (Jupiter, Raydium, Meteora, etc.), and red-flag language.

Red Flags & Penalties

The scoring engine detects suspicious patterns (up to -10 points): brand new wallets with extreme transaction volume, single-protocol bots, spam-speed transactions, instant token dumps, and heavy SOL drains.

Task-Specific Profiles

Different tasks require different trust signals. Use the task parameter to score agents against optimized weight profiles:

default defi_execution content_creation social_management trading data_analysis high_value

Data Sources

All data is live — no simulated or cached scores. Sources: FairScale API (Pro tier, 1-year behavioral window, daily rescoring), SAID Protocol (on-chain PDA discovery via Helius), ERC-8004 Solana Agent Registry (Helius DAS), SATI Token-2022 NFTs, ClawKey/VeryAI human verification.

API Reference

Base URL: https://fairscale-reputation-api-production.up.railway.app

All endpoints return JSON. No authentication required for public endpoints. Rate limits apply.

GET /v1/score

Score any Solana wallet. Returns full breakdown with all six pillar scores, percentiles, red flags, attestation data, and trust summary.

GET /v1/score?wallet=8oPy1bQ...&task=defi_execution

Parameters:

wallet (required) — Solana wallet address

task (optional) — Task profile for custom weight distribution

GET /v1/trust-gate

Single-call allow/deny decision for protocols. Designed for embedding in transaction flows.

GET /v1/trust-gate?wallet=8oPy1bQ...&min_score=50&task=defi_execution&require_verification=true

Parameters:

wallet (required) — Wallet to check

min_score — Minimum score to allow (default: 40)

task — Task-specific weight profile

require_verification — Require at least one protocol registration (default: true)

require_no_flags — Reject agents with red flags (default: false)

Example response:

{
  "decision": "allow",
  "wallet": "8oPy1bQ...",
  "fairscore": 68,
  "recommendation": "Recommended",
  "reasons": ["Score 68 >= minimum 50", "Verified on ERC-8004"],
  "red_flags": [],
  "verification": { "said": true, "erc8004": true, "sati": false, "clawkey": false }
}

GET /v1/directory

Query the full agent directory with filters, pagination, and sorting.

GET /v1/directory?page=1&limit=25&sort=agent_fairscore&source=both&min_score=50

Parameters: page, limit (max 100), sort (any pillar name), source (said/8004/both), search, recommendation, min_score, verified_only, has_flags

Integration Example

Embed trust-gating in your protocol's transaction flow:

// Before executing an agent's trade request:
const res = await fetch(
  'https://fairscale-reputation-api-production.up.railway.app' +
  '/v1/trust-gate?wallet=' + agentWallet +
  '&min_score=50&task=defi_execution&require_verification=true'
);
const { decision, fairscore, reasons } = await res.json();

if (decision === 'allow') {
  // Execute the agent's request
  await executeAgentTrade(agentWallet, tradeParams);
} else {
  // Reject or require additional verification
  console.log('Agent rejected:', reasons);
}
